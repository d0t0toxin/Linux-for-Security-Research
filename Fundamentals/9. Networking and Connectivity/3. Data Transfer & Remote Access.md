# Linux `curl` Command

The `curl` command in Linux is a powerful tool used to transfer data to or from a server using various protocols such as HTTP, HTTPS, FTP, and more. It is widely used for testing APIs, downloading files, and automating network tasks.

* * *

## Syntax

```bash
curl [options] [URL]
```

- `URL` is the target server's address.

* * *

## Basic Usage

### 1\. Fetch a Web Page

```bash
curl https://example.com
```

- Retrieves the content of the web page and displays it in the terminal.

### 2\. Save Output to a File

```bash
curl -o example.html https://example.com
```

- `-o` specifies the output filename.

### 3\. Follow Redirects

```bash
curl -L https://example.com
```

- `-L` follows HTTP redirects.

### 4\. Show Only HTTP Headers

```bash
curl -I https://example.com
```

- `-I` fetches headers only (HEAD request).

### 5\. Download Multiple Files

```bash
curl -O https://example.com/file1.txt -O https://example.com/file2.txt
```

- `-O` saves files with their original names.

### 6\. Send Data with POST Request

```bash
curl -X POST -d "username=user&password=pass" https://example.com/login
```

- `-X POST` specifies POST request.
    
- `-d` sends form data.
    

### 7\. Send JSON Data

```bash
curl -X POST -H "Content-Type: application/json" -d '{"name":"John","age":30}' https://example.com/api
```

- `-H` sets header.
    
- `-d` sends JSON payload.
    

### 8\. Add Custom Headers

```bash
curl -H "Authorization: Bearer TOKEN" https://example.com/api
```

- Adds custom headers for API requests.

### 9\. Show Progress and Download Bar

```bash
curl -O -# https://example.com/file.zip
```

- `-#` shows a progress bar.

### 10\. Limit Download Speed

```bash
curl --limit-rate 100K -O https://example.com/file.zip
```

- Limits download to 100 KB/s.

### 11\. Resume Interrupted Downloads

```bash
curl -C - -O https://example.com/file.zip
```

- `-C -` resumes download from where it stopped.

### 12\. FTP Upload

```bash
curl -T localfile.txt ftp://ftp.example.com/ --user username:password
```

- Uploads file to FTP server.

### 13\. Use Proxy

```bash
curl -x http://proxyserver:8080 https://example.com
```

- Sends request via proxy.

### 14\. Silent Mode

```bash
curl -s https://example.com
```

- `-s` suppresses progress and error messages.

### 15\. Verbose Mode

```bash
curl -v https://example.com
```

- `-v` shows detailed request/response information.

* * *

## Common Options Summary

| Option | Description |
| --- | --- |
| `-o <file>` | Save output to a file |
| `-O` | Save file with original name |
| `-L` | Follow redirects |
| `-I` | Fetch headers only |
| `-d <data>` | Send data with POST |
| `-X <method>` | Specify HTTP method (GET, POST, PUT, DELETE, etc.) |
| `-H <header>` | Add custom headers |
| `-s` | Silent mode |
| `-v` | Verbose mode |
| `-#` | Progress bar |
| `--limit-rate <rate>` | Limit download/upload speed |
| `-C -` | Resume download |
| `-u user:password` | Provide username and password |
| `-x <proxy>` | Use proxy server |

* * *

## Notes

- `curl` supports a wide range of protocols: HTTP, HTTPS, FTP, FTPS, SCP, SFTP, LDAP, and more.
    
- Useful for API testing, downloading/uploading files, and debugging network requests.
    
- Install if missing:
    
    - Debian/Ubuntu: `sudo apt install curl`
        
    - RHEL/CentOS: `sudo yum install curl`

# Linux `wget` Command

The `wget` command in Linux is a non-interactive network downloader that retrieves files from the web using HTTP, HTTPS, and FTP protocols. It's widely used for downloading files, recursive downloads, and automating downloads in scripts.

* * *

## Syntax

```bash
wget [options] URL
```

- `URL` is the target file or webpage.

* * *

## Basic Usage

### 1\. Download a Single File

```bash
wget https://example.com/file.zip
```

- Downloads `file.zip` to the current directory.

### 2\. Save File with a Different Name

```bash
wget -O myfile.zip https://example.com/file.zip
```

- `-O` specifies the output filename.

### 3\. Continue an Interrupted Download

```bash
wget -c https://example.com/file.zip
```

- `-c` resumes download from where it stopped.

### 4\. Download in Background

```bash
wget -b https://example.com/file.zip
```

- `-b` downloads file in the background.

### 5\. Limit Download Speed

```bash
wget --limit-rate=100k https://example.com/file.zip
```

- Limits download speed to 100 KB/s.

### 6\. Recursive Download of a Website

```bash
wget -r https://example.com
```

- `-r` downloads the entire website recursively.

### 7\. Download Only Certain File Types

```bash
wget -r -A pdf,docx https://example.com
```

- `-A` allows only files with specified extensions.

### 8\. Mirror a Website

```bash
wget --mirror -p --convert-links -P ./localdir https://example.com
```

- `--mirror` enables recursive download with timestamping.
    
- `-p` downloads all page requisites (images, CSS).
    
- `--convert-links` converts links for local viewing.
    
- `-P ./localdir` saves files to `./localdir`.
    

### 9\. Download from FTP with Username and Password

```bash
wget --ftp-user=username --ftp-password=password ftp://ftp.example.com/file.txt
```

- Authenticates and downloads file from FTP server.

### 10\. Quiet Mode

```bash
wget -q https://example.com/file.zip
```

- `-q` suppresses output.

### 11\. Verbose Mode

```bash
wget -v https://example.com/file.zip
```

- `-v` displays detailed download information.

### 12\. Retry on Failure

```bash
wget -t 5 https://example.com/file.zip
```

- `-t 5` retries download 5 times if it fails.

* * *

## Common Options Summary

| Option | Description |
| --- | --- |
| `-O <file>` | Save file with a specific name |
| `-c` | Continue interrupted download |
| `-b` | Download in background |
| `--limit-rate=<rate>` | Limit download speed |
| `-r` | Recursive download |
| `-A <list>` | Accept only specified file types |
| `--mirror` | Mirror a website |
| `-p` | Download all page requisites |
| `--convert-links` | Convert links for local browsing |
| `-P <directory>` | Save files to specified directory |
| `--ftp-user` | FTP username |
| `--ftp-password` | FTP password |
| `-q` | Quiet mode |
| `-v` | Verbose mode |
| `-t <number>` | Number of retries on failure |

* * *

## Notes

- `wget` is non-interactive and ideal for scripts and automated tasks.
    
- Supports HTTP, HTTPS, and FTP protocols.
    
- Can handle recursive downloads, mirroring, and file filtering.
    
- Install if missing:
    
    - Debian/Ubuntu: `sudo apt install wget`
        
    - RHEL/CentOS: `sudo yum install wget`

# Linux `ftp` Command

The `ftp` command in Linux is a client program used to transfer files between your local machine and a remote FTP server. It supports both uploading and downloading files using the File Transfer Protocol (FTP).

* * *

## Syntax

```bash
ftp [options] [host]
```

- `host` is the FTP server address.

* * *

## Basic Usage

### 1\. Connect to an FTP Server

```bash
ftp ftp.example.com
```

- Prompts for username and password to log in.

### 2\. Login with Username and Password

```bash
ftp> open ftp.example.com
Name (ftp.example.com:user): username
Password: ******
```

- `open` connects to the FTP server.
    
- Enter your credentials when prompted.
    

### 3\. List Files on Remote Server

```bash
ftp> ls
```

- Displays files and directories in the current remote directory.

### 4\. Change Remote Directory

```bash
ftp> cd /path/to/directory
```

- Changes to the specified directory on the remote server.

### 5\. Download a File from Remote Server

```bash
ftp> get remote_file.txt
```

- Downloads `remote_file.txt` to the local machine.

### 6\. Download Multiple Files

```bash
ftp> mget *.txt
```

- Downloads all `.txt` files in the current remote directory.

### 7\. Upload a File to Remote Server

```bash
ftp> put local_file.txt
```

- Uploads `local_file.txt` from local machine to the remote server.

### 8\. Upload Multiple Files

```bash
ftp> mput *.txt
```

- Uploads all `.txt` files from local directory to remote server.

### 9\. Create a Directory on Remote Server

```bash
ftp> mkdir newdir
```

- Creates a directory named `newdir` on the FTP server.

### 10\. Change Local Directory

```bash
ftp> lcd /local/path
```

- Changes the local directory to `/local/path`.

### 11\. Delete a Remote File

```bash
ftp> delete remote_file.txt
```

- Deletes `remote_file.txt` from the server.

### 12\. Rename a Remote File

```bash
ftp> rename oldname.txt newname.txt
```

- Renames `oldname.txt` to `newname.txt` on the server.

### 13\. Switch to Binary Mode (for non-text files)

```bash
ftp> binary
```

- Recommended for images, videos, or compressed files.

### 14\. Switch to ASCII Mode (for text files)

```bash
ftp> ascii
```

- Default mode for text files.

### 15\. Exit FTP

```bash
ftp> bye
```

- Closes the connection and exits the FTP client.

* * *

## Common Commands Summary

| Command | Description |
| --- | --- |
| `open <host>` | Connect to FTP server |
| `user <username>` | Login with username |
| `ls` | List files on remote server |
| `cd <dir>` | Change remote directory |
| `lcd <dir>` | Change local directory |
| `get <file>` | Download a file |
| `mget <files>` | Download multiple files |
| `put <file>` | Upload a file |
| `mput <files>` | Upload multiple files |
| `mkdir <dir>` | Create remote directory |
| `delete <file>` | Delete remote file |
| `rename <old> <new>` | Rename remote file |
| `binary` | Switch to binary mode |
| `ascii` | Switch to ASCII mode |
| `bye` / `quit` | Exit FTP client |

* * *

## Notes

- FTP transmits data unencrypted. For secure transfers, consider `sftp` or `ftps`.
    
- Use `lcd` to navigate local files while connected.
    
- Multiple files can be downloaded or uploaded using wildcards (`*`).
    
- Install if missing:
    
    - Debian/Ubuntu: `sudo apt install ftp`
        
    - RHEL/CentOS: `sudo yum install ftp`

# Linux `scp` Command

The `scp` (secure copy) command in Linux is used to securely transfer files and directories between local and remote systems over SSH. It ensures that the data is encrypted during transmission.

* * *

## Syntax

```bash
scp [options] source destination
```

- `source`: File or directory to copy.
    
- `destination`: Target location (local or remote).
    
- Remote paths follow the format: `user@host:/path/to/destination`
    

* * *

## Basic Usage

### 1\. Copy a Local File to a Remote Server

```bash
scp localfile.txt user@remotehost:/home/user/
```

- Copies `localfile.txt` to `/home/user/` on the remote host.

### 2\. Copy a Remote File to Local Machine

```bash
scp user@remotehost:/home/user/remotefile.txt ./
```

- Downloads `remotefile.txt` from remote host to current local directory.

### 3\. Copy a Directory Recursively

```bash
scp -r localdir user@remotehost:/home/user/
```

- `-r` copies entire directory `localdir` including all subdirectories and files.

### 4\. Specify a Port Other Than Default (22)

```bash
scp -P 2222 localfile.txt user@remotehost:/home/user/
```

- `-P 2222` uses port 2222 for SSH.

### 5\. Copy Multiple Files to Remote Server

```bash
scp file1.txt file2.txt user@remotehost:/home/user/
```

- Copies both `file1.txt` and `file2.txt` to the remote directory.

### 6\. Use Compression During Transfer

```bash
scp -C largefile.zip user@remotehost:/home/user/
```

- `-C` enables compression for faster transfer of large files.

### 7\. Limit Bandwidth During Transfer

```bash
scp -l 500 localfile.txt user@remotehost:/home/user/
```

- `-l 500` limits the bandwidth to 500 Kbit/s.

### 8\. Copy from Remote to Remote Host

```bash
scp user1@host1:/path/file.txt user2@host2:/path/
```

- Copies a file directly between two remote hosts.

### 9\. Verbose Mode (Debugging)

```bash
scp -v localfile.txt user@remotehost:/home/user/
```

- Displays detailed information about the transfer.

### 10\. Preserve File Attributes

```bash
scp -p localfile.txt user@remotehost:/home/user/
```

- `-p` preserves modification times, access times, and modes.

* * *

## Common Options Summary

| Option | Description |
| --- | --- |
| `-r` | Copy directories recursively |
| `-P <port>` | Specify SSH port (note uppercase P) |
| `-C` | Enable compression |
| `-l <limit>` | Limit bandwidth in Kbit/s |
| `-v` | Verbose mode |
| `-p` | Preserve file attributes |
| `-q` | Quiet mode (suppress progress) |

* * *

## Notes

- `scp` relies on SSH, so the SSH service must be running on the remote host.
    
- Use `-r` for directories and `-C` for compressing large files.
    
- Bandwidth limiting (`-l`) is useful on slow networks.
    
- Install if missing:
    
    - Debian/Ubuntu: `sudo apt install openssh-client`
        
    - RHEL/CentOS: `sudo yum install openssh-clients`

# Linux `sftp` Command

The `sftp` (Secure File Transfer Protocol) command in Linux is used to securely transfer files between a local and remote system over SSH. It provides an interactive interface similar to `ftp` but encrypted.

* * *

## Syntax

```bash
sftp [options] [user@]host
```

- `user@host` specifies the remote username and host.

* * *

## Basic Usage

### 1\. Connect to a Remote Host

```bash
sftp user@remotehost
```

- Prompts for the password and opens an interactive SFTP session.

### 2\. Upload a File to Remote Server

```bash
sftp> put localfile.txt
```

- Uploads `localfile.txt` from the local machine to the current directory on the remote server.

### 3\. Download a File from Remote Server

```bash
sftp> get remotefile.txt
```

- Downloads `remotefile.txt` from remote server to local machine.

### 4\. Upload Multiple Files

```bash
sftp> mput *.txt
```

- Uploads all `.txt` files from local directory to remote server.

### 5\. Download Multiple Files

```bash
sftp> mget *.txt
```

- Downloads all `.txt` files from remote directory to local machine.

### 6\. Change Remote Directory

```bash
sftp> cd /path/to/remote/dir
```

- Changes the current directory on the remote server.

### 7\. Change Local Directory

```bash
sftp> lcd /path/to/local/dir
```

- Changes the local working directory.

### 8\. List Files on Remote Server

```bash
sftp> ls
```

- Lists files and directories in the current remote directory.

### 9\. Display Remote Directory Path

```bash
sftp> pwd
```

- Shows the current directory on the remote server.

### 10\. Display Local Directory Path

```bash
sftp> lpwd
```

- Shows the current local directory.

### 11\. Create Directory on Remote Server

```bash
sftp> mkdir newdir
```

- Creates a new directory named `newdir` on remote server.

### 12\. Remove a File on Remote Server

```bash
sftp> rm remotefile.txt
```

- Deletes `remotefile.txt` on remote server.

### 13\. Remove a Directory on Remote Server

```bash
sftp> rmdir dir
```

- Deletes directory `dir` on remote server.

### 14\. Rename a File on Remote Server

```bash
sftp> rename oldname.txt newname.txt
```

- Renames file `oldname.txt` to `newname.txt`.

### 15\. Exit SFTP Session

```bash
sftp> exit
```

- Closes the connection and exits the session.

* * *

## Common Commands Summary

| Command | Description |
| --- | --- |
| `put <file>` | Upload a file |
| `mput <files>` | Upload multiple files |
| `get <file>` | Download a file |
| `mget <files>` | Download multiple files |
| `cd <dir>` | Change remote directory |
| `lcd <dir>` | Change local directory |
| `ls` | List files on remote server |
| `pwd` | Show remote current directory |
| `lpwd` | Show local current directory |
| `mkdir <dir>` | Create remote directory |
| `rm <file>` | Delete remote file |
| `rmdir <dir>` | Delete remote directory |
| `rename <old> <new>` | Rename remote file |
| `exit` / `quit` | Exit SFTP session |

* * *

## Notes

- `sftp` encrypts all data, including credentials, unlike `ftp`.
    
- Supports both interactive and batch modes.
    
- Can be used to automate secure file transfers in scripts.
    
- Install if missing:
    
    - Debian/Ubuntu: `sudo apt install openssh-client`
        
    - RHEL/CentOS: `sudo yum install openssh-clients`

# Linux `ssh` Command

The `ssh` (Secure Shell) command in Linux is used to securely connect to remote machines over an encrypted network connection. It is widely used for remote administration, command execution, file transfer, and tunneling.

* * *

## What is SSH?

SSH provides:

- Encrypted communication
    
- Secure remote login
    
- Secure command execution
    
- Port forwarding and tunneling
    
- Authentication using passwords or SSH keys
    

Default port: **22**

* * *

## Syntax

```bash
ssh [options] [user@]host [command]
```

- `user` – Remote username (optional)
    
- `host` – Hostname or IP address
    
- `command` – Command to execute remotely (optional)
    

* * *

## Basic Usage

### 1\. Connect to a Remote Server

```bash
ssh user@remotehost
```

- Connects to `remotehost` as `user`.

### 2\. Connect Using IP Address

```bash
ssh user@192.168.1.10
```

### 3\. Connect Using Default User

```bash
ssh remotehost
```

- Uses the current local username.

### 4\. Specify a Custom Port

```bash
ssh -p 2222 user@remotehost
```

- Uses SSH port `2222` instead of 22.

* * *

## Executing Remote Commands

### 5\. Run a Single Command Remotely

```bash
ssh user@remotehost ls -l /var/www
```

- Executes command and exits.

### 6\. Run Multiple Commands

```bash
ssh user@remotehost "uptime && df -h"
```

* * *

## SSH Key-Based Authentication

### 7\. Generate SSH Key Pair

```bash
ssh-keygen -t rsa -b 4096
```

- Creates a public/private key pair.

### 8\. Copy Public Key to Remote Server

```bash
ssh-copy-id user@remotehost
```

- Enables passwordless login.

### 9\. Login Using SSH Key

```bash
ssh user@remotehost
```

* * *

## File Transfer Using SSH

### 10\. Copy Files Securely (scp)

```bash
scp file.txt user@remotehost:/home/user/
```

### 11\. Secure File Transfer (sftp)

```bash
sftp user@remotehost
```

* * *

## SSH Tunneling & Port Forwarding

### 12\. Local Port Forwarding

```bash
ssh -L 8080:localhost:80 user@remotehost
```

- Access remote port 80 via `localhost:8080`.

### 13\. Remote Port Forwarding

```bash
ssh -R 9090:localhost:3000 user@remotehost
```

- Exposes local port 3000 to remote port 9090.

### 14\. Dynamic Port Forwarding (SOCKS Proxy)

```bash
ssh -D 1080 user@remotehost
```

- Creates a SOCKS proxy on port 1080.

* * *

## Common SSH Options

| Option | Description |
| --- | --- |
| `-p <port>` | Specify SSH port |
| `-i <key>` | Use specific private key |
| `-v` | Verbose/debug mode |
| `-C` | Enable compression |
| `-L` | Local port forwarding |
| `-R` | Remote port forwarding |
| `-D` | Dynamic SOCKS proxy |
| `-N` | Do not execute remote command |
| `-T` | Disable pseudo-terminal allocation |

* * *

## SSH Configuration File

### 15\. SSH Config File Location

```bash
~/.ssh/config
```

### 16\. Example SSH Config

```text
Host myserver
    HostName 192.168.1.10
    User ubuntu
    Port 2222
    IdentityFile ~/.ssh/id_rsa
```

### Connect Using Alias

```bash
ssh myserver
```

* * *

## Security Best Practices

- Use SSH keys instead of passwords
    
- Disable root login
    
- Change default SSH port
    
- Use firewalls and fail2ban
    
- Set proper permissions:
    

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_rsa
```

* * *

## Troubleshooting

### Test Connection Verbosely

```bash
ssh -vvv user@remotehost
```

### Check SSH Service

```bash
sudo systemctl status sshd
```

* * *

## Installation

- Debian/Ubuntu:

```bash
sudo apt install openssh-client openssh-server
```

- RHEL/CentOS:

```bash
sudo yum install openssh openssh-server
```

* * *

## Notes

- SSH encrypts all traffic including passwords
    
- Used for remote administration, DevOps, cloud, and servers
    
- Backbone of many Linux automation workflows

# Linux `rsync` Command

The `rsync` command in Linux is a fast, versatile file-copying and synchronization tool. It efficiently transfers and syncs files between local directories, remote systems, or across SSH while minimizing data transfer by sending only differences.

* * *

## What is `rsync`?

`rsync` is used for:

- Local and remote file synchronization
    
- Incremental backups
    
- Efficient copying of large directories
    
- Mirroring directories
    
- Backup automation
    

Key features:

- Delta transfer algorithm (copies only changes)
    
- Preserves permissions, ownership, timestamps
    
- Works over SSH
    
- Resume interrupted transfers
    

* * *

## Syntax

```bash
rsync [options] source destination
```

- `source` – File or directory to copy
    
- `destination` – Target location (local or remote)
    

Remote format:

```bash
user@host:/path
```

* * *

## Basic Usage

### 1\. Copy a File Locally

```bash
rsync file.txt /backup/
```

### 2\. Copy a Directory Locally

```bash
rsync -r mydir /backup/
```

* * *

## Most Commonly Used Option: `-a`

### 3\. Archive Mode (Recommended)

```bash
rsync -a source/ destination/
```

`-a` (archive) includes:

- `-r` recursive
    
- `-l` copy symlinks
    
- `-p` preserve permissions
    
- `-t` preserve timestamps
    
- `-g` preserve group
    
- `-o` preserve owner
    

* * *

## Trailing Slash (`/`) Behavior (VERY IMPORTANT)

```bash
rsync -a dir1 dir2
```

Copies **dir1** inside dir2 → `dir2/dir1`

```bash
rsync -a dir1/ dir2
```

Copies **contents of dir1** into dir2

* * *

## Remote Transfers (Over SSH)

### 4\. Copy Local → Remote

```bash
rsync -av file.txt user@remote:/home/user/
```

### 5\. Copy Remote → Local

```bash
rsync -av user@remote:/home/user/file.txt ./
```

### 6\. Copy Directory Over SSH

```bash
rsync -avz mydir/ user@remote:/backup/mydir/
```

* * *

## Compression & Speed

### 7\. Enable Compression

```bash
rsync -avz source/ destination/
```

- `-z` compresses data during transfer

### 8\. Limit Bandwidth

```bash
rsync -av --bwlimit=500 source/ destination/
```

- Limits bandwidth to 500 KB/s

* * *

## Delete Files (Mirroring)

### 9\. Mirror Source to Destination

```bash
rsync -av --delete source/ destination/
```

- Deletes files in destination that no longer exist in source

**Use carefully – data loss possible**

* * *

## Dry Run (Preview Changes)

### 10\. See What Will Happen (No Changes)

```bash
rsync -av --dry-run source/ destination/
```

* * *

## Excluding & Including Files

### 11\. Exclude Files or Directories

```bash
rsync -av --exclude="*.log" source/ destination/
```

### 12\. Multiple Excludes

```bash
rsync -av --exclude="node_modules" --exclude="*.tmp" source/ destination/
```

### 13\. Use Exclude File

```bash
rsync -av --exclude-from=exclude.txt source/ destination/
```

* * *

## Resume Interrupted Transfers

### 14\. Resume Partial Transfers

```bash
rsync -av --partial source/ destination/
```

### 15\. Resume Large Files Efficiently

```bash
rsync -av --partial --progress source/ destination/
```

* * *

## Progress & Verbosity

### 16\. Show Progress

```bash
rsync -av --progress source/ destination/
```

### 17\. Human-Readable Output

```bash
rsync -avh source/ destination/
```

* * *

## Backup & Versioning

### 18\. Create Incremental Backups

```bash
rsync -av --backup --backup-dir=/backups/old source/ destination/
```

* * *

## Using a Custom SSH Port or Key

### 19\. Custom SSH Port

```bash
rsync -av -e "ssh -p 2222" source/ user@remote:/backup/
```

### 20\. Custom SSH Key

```bash
rsync -av -e "ssh -i ~/.ssh/id_rsa" source/ user@remote:/backup/
```

* * *

## Common Options Summary

| Option | Description |
| --- | --- |
| `-a` | Archive mode (most used) |
| `-v` | Verbose output |
| `-z` | Compress data |
| `-r` | Recursive copy |
| `--delete` | Delete extra files in destination |
| `--dry-run` | Preview changes |
| `--exclude` | Exclude files/directories |
| `--progress` | Show progress |
| `--bwlimit` | Limit bandwidth |
| `-e ssh` | Use SSH for transfer |

* * *

## Real-World Examples

### Backup Home Directory

```bash
rsync -av --delete /home/user/ /mnt/backup/home/
```

### Sync Website to Server

```bash
rsync -avz ./site/ user@server:/var/www/html/
```

### Daily Cron Backup

```bash
rsync -av --delete /data/ /backup/data/
```

## Notes & Best Practices

- Always use `--dry-run` before `--delete`
    
- Prefer `-a` for backups
    
- Trailing slash matters
    
- `rsync` is safer and faster than `cp` or `scp` for large data
