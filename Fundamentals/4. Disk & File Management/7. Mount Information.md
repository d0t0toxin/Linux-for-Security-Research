# `mountpoint` Command in Linux

`mountpoint` is a Linux command-line utility used to **check if a directory or path is a mount point** for a filesystem. It is a simple, focused tool often used in scripts for validation.

* * *

## 1\. What is `mountpoint`?

- Checks if a directory is a **mount point** (i.e., a mounted filesystem root).
    
- Returns an **exit status** that can be used in scripts.
    
- Does **not mount or unmount** filesystems.
    
- Part of the `util-linux` package (usually pre-installed).
    

* * *

## 2\. Syntax

```bash
mountpoint [options] <directory>
```

- `<directory>`: The path to check if it is a mount point.

Example:

```bash
mountpoint /mnt/data
```

* * *

## 3\. Exit Status

- `0`: Directory **is a mount point**
    
- `1`: Directory **exists but is not a mount point**
    
- `2`: Directory **does not exist**
    

This is useful for **shell scripts** to check mount status programmatically.

* * *

## 4\. Common Options

| Option | Description |
| --- | --- |
| `-q` | Quiet mode (no output, only exit status) |
| `-d` | Display the directory's device if it is a mount point |
| `-n` | Do not resolve symlinks |
| `-x <fstype>` | Only consider mount points of the specified filesystem type |

* * *

## 5\. Basic Examples

### Check if a directory is a mount point

```bash
mountpoint /mnt/data
```

Output:

```
/mnt/data is a mountpoint
```

### Quiet check in scripts

```bash
mountpoint -q /mnt/data && echo "Mounted" || echo "Not mounted"
```

### Check mount point of a directory without resolving symlinks

```bash
mountpoint -n /mnt/data
```

### Check only for specific filesystem type

```bash
mountpoint -x ext4 /mnt/data
```

### Display the device of the mount point

```bash
mountpoint -d /mnt/data
```

Output:

```
/mnt/data is mounted on /dev/sdb1
```

* * *

## 6\. Example Workflow

```bash
# 1. Create a mount directory
sudo mkdir /mnt/data

# 2. Mount a partition
sudo mount /dev/sdb1 /mnt/data

# 3. Check if it is mounted
mountpoint /mnt/data
# Output: /mnt/data is a mountpoint

# 4. Script example
if mountpoint -q /mnt/data; then
    echo "Partition is mounted"
else
    echo "Partition is not mounted"
fi
```

* * *

## 7\. Tips and Warnings

1.  `mountpoint` does **not mount or unmount** filesystems; use `mount`/`umount` for that.
    
2.  Useful in **shell scripts** for automation and validation.
    
3.  Works with **all filesystem types** supported by the kernel.
    
4.  `-q` is ideal when only the exit status is needed in scripts.

# `findmnt` Command in Linux

`findmnt` is a Linux command-line utility used to **find mounted filesystems and their mount points**. It is a powerful tool for inspecting mounts and automating checks in scripts.

* * *

## 1\. What is `findmnt`?

- Displays **mount points, devices, and filesystem types**.
    
- Can filter output based on device, mount point, or filesystem type.
    
- Useful in **scripts, debugging, or system monitoring**.
    
- Part of the `util-linux` package (usually pre-installed).
    

* * *

## 2\. Syntax

```bash
findmnt [options] [<source> | <target> | <fstype>]
```

- `<source>`: Device or filesystem source, e.g., `/dev/sda1`
    
- `<target>`: Mount point, e.g., `/mnt/data`
    
- `<fstype>`: Filesystem type, e.g., `ext4`
    

Example:

```bash
findmnt /mnt/data
```

* * *

## 3\. Common Options

| Option | Description |
| --- | --- |
| `-l` | List all mounts in **list format** |
| `-t <fstype>` | Show only filesystems of the specified type |
| `-S <source>` | Show only the specified source device |
| `-D <target>` | Show only the specified mount point |
| `-n` | Do not print the heading row |
| `-r` | Show only **real** mount points (skip pseudo filesystems like `proc`) |
| `-o <columns>` | Show only specific columns, e.g., `TARGET,SOURCE,FSTYPE` |
| `-a` | Print all available fields |
| `-J` | Output in JSON format |
| `-T` | Display the filesystem **table in tree format** |

* * *

## 4\. Basic Examples

### Find mount point for a directory

```bash
findmnt /mnt/data
```

Output:

```
TARGET    SOURCE    FSTYPE OPTIONS
/mnt/data /dev/sdb1 ext4   rw,relatime
```

### Find all mounts

```bash
findmnt
```

### List mounts in list format

```bash
findmnt -l
```

### Show only specific filesystem type (ext4)

```bash
findmnt -t ext4
```

### Show only a specific device

```bash
findmnt -S /dev/sdb1
```

### Show only specific columns

```bash
findmnt -o TARGET,SOURCE,FSTYPE /mnt/data
```

Output:

```
/mnt/data /dev/sdb1 ext4
```

### Display in tree format

```bash
findmnt -T /mnt/data
```

### Output in JSON

```bash
findmnt -J
```

* * *

## 5\. Example Workflow

```bash
# 1. Mount a device
sudo mount /dev/sdb1 /mnt/data

# 2. Verify mount using findmnt
findmnt /mnt/data

# 3. Check all ext4 filesystems
findmnt -t ext4

# 4. Get only target and source columns
findmnt -o TARGET,SOURCE

# 5. Use in scripts (quiet check)
if findmnt -n /mnt/data > /dev/null; then
    echo "Device is mounted"
else
    echo "Device is not mounted"
fi
```

* * *

## 6\. Tips and Warnings

1.  `findmnt` **does not mount or unmount**; use `mount`/`umount` for that.
    
2.  Can filter results by **source, target, or filesystem type**.
    
3.  Useful for **scripts to check mount points and verify mounts**.
    
4.  `-J` JSON output is useful for automation and parsing.
    
5.  Combine with `grep` or `awk` for custom reporting.
