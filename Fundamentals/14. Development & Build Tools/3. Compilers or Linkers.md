# GCC Command in Linux

## Introduction

GCC (GNU Compiler Collection) is a powerful compiler used to compile programs written in C, C++, and other languages. It is widely used for building software on Linux systems.

## Basic Syntax

```
gcc [options] source_file -o output_file
```

- **source_file**: The source code file to compile (e.g., main.c)
- **output_file**: The name of the executable generated (optional)

## Basic Examples

### Example 1: Simple Compilation

```
gcc main.c -o main
```

This compiles `main.c` into an executable named `main`.

### Example 2: Compilation without Specifying Output

```
gcc main.c
```

This produces an executable named `a.out` by default.

### Example 3: Compiling Multiple Files

```
gcc main.c utils.c -o program
```

This compiles `main.c` and `utils.c` into a single executable named `program`.

## Common GCC Options with Examples

### 1\. -o

Specifies the name of the output file.

```
gcc main.c -o myprogram
```

### 2\. -c

Compiles source files into object files without linking.

```
gcc -c main.c
```

This produces `main.o`.

### 3\. -Wall

Enables all common warning messages.

```
gcc -Wall main.c -o main
```

### 4\. -Werror

Treats all warnings as errors.

```
gcc -Wall -Werror main.c -o main
```

### 5\. -g

Includes debugging information in the executable.

```
gcc -g main.c -o main
```

### 6\. -O, -O1, -O2, -O3

Specifies optimization levels:

- `-O` or `-O1`: Basic optimization
- `-O2`: More optimization
- `-O3`: Maximum optimization

```
gcc -O2 main.c -o main
```

### 7\. -I

Specifies the directory for header files.

```
gcc main.c -I./include -o main
```

### 8\. -L

Specifies the directory for library files.

```
gcc main.c -L./lib -lmylib -o main
```

### 9\. -l

Links a library.

```
gcc main.c -lm  # Links math library
```

### 10\. -E

Preprocesses the source file and outputs the result.

```
gcc -E main.c -o main.i
```

### 11\. -S

Compiles the source file into assembly code.

```
gcc -S main.c -o main.s
```

### 12\. -static

Generates a statically linked executable.

```
gcc main.c -static -o main
```

### 13\. -shared

Generates a shared library.

```
gcc -shared -o libmylib.so utils.c
```

### 14\. -fPIC

Generates position-independent code (used for shared libraries).

```
gcc -fPIC -c utils.c
```

### 15\. -D

Defines a macro.

```
gcc -DDEBUG main.c -o main
```

### 16\. -std

Specifies the language standard.

```
gcc -std=c99 main.c -o main
```

### 17\. -pthread

Enables multithreading with pthreads.

```
gcc main.c -pthread -o main
```

### 18\. --help

Displays all GCC options.

```
gcc --help
```

## Advanced Concepts

### Combining Options

```
gcc -Wall -Werror -O2 -g main.c utils.c -o program -lm
```

### Compiling for Debugging and Optimization

```
gcc -O2 -g main.c -o main
```

### Generating Assembly and Object Files

```
gcc -S main.c -o main.s
gcc -c main.c -o main.o
```

## Conclusion

GCC is a versatile and essential compiler in Linux development. Understanding its options allows precise control over compilation, optimization, linking, and debugging processes.
